Guía de Trabajo en Equipo con GitHub

¿Qué es GitHub?

GitHub es una plataforma de desarrollo colaborativo para gestionar proyectos y control de versiones usando Git. Facilita el trabajo en equipo, permite gestionar el código fuente y ofrece herramientas para revisión y despliegue.

Conceptos Clave

  • Repositorio: Carpeta en la que se almacena el proyecto con su historial de cambios.
  • Commit: Registro de cambios hechos en el proyecto.
  • Rama (Branch): Línea de trabajo independiente para hacer cambios sin afectar la rama principal.
  • Pull Request: Solicitud de integración de una rama a otra para revisión de cambios.
  • Merge: Acción de combinar cambios de una rama en otra.

Configuración del Proyecto y Permisos

  1. Crear el repositorio en GitHub.
  2. Agregar colaboradores en Settings > Collaborators.
  3. Configurar permisos para que otros usuarios puedan realizar cambios.

Comandos Básicos para Trabajo en Equipo

Los siguientes comandos son esenciales para que un usuario pueda modificar el proyecto y otro pueda deshacer los cambios:

1. Clonar el Repositorio

git clone <URL_del_Repositorio>

2. Crear una Rama para Modificar el Proyecto

Permite a cada colaborador trabajar en su propia línea de trabajo sin afectar la rama principal.

git checkout -b nombre-de-la-rama

3. Realizar Commit y Subir Cambios

                    git add .
                    git commit -m "Descripción de los cambios"
                    git push origin nombre-de-la-rama

4. Crear un Pull Request

Al subir los cambios, el colaborador puede abrir un pull request en GitHub para que otros revisen y aprueben los cambios antes de fusionarlos.

5. Deshacer Cambios

Si otro colaborador quiere revertir el commit de un compañero, puede usar:

                    # Para revertir el último commit sin borrar el trabajo
                    git revert HEAD

                    # Para revertir completamente a un commit anterior
                    git reset --hard <commit_ID>

Mejores Prácticas de Colaboración

  • Usar ramas para cada funcionalidad o tarea.
  • Hacer commits frecuentes y con mensajes descriptivos.
  • Revisar y aprobar pull requests antes de hacer merge.
  • Comunicarse constantemente con el equipo para resolver dudas y conflictos.
